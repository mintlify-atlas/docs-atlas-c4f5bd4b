## Prerequisites

Before you start, make sure you have:

- **Node.js** v18 or later
- **Microsoft SQL Server** (e.g., SQL Server Express) running locally or remotely
- **SMTP access** for email notifications (e.g., a Gmail account with an app password)

## Steps

<Steps>
  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/MariaJose1812/Api-Tickets.git
    cd Api-Tickets
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    npm install
    ```

    This installs Express, mssql, jsonwebtoken, bcrypt, nodemailer, and other required packages.
  </Step>

  <Step title="Configure environment variables">
    Create a `.env` file in the project root. At minimum, set your database credentials, JWT secret, and SMTP settings:

    ```ini .env
    PORT=3000

    SQL_SERVER=localhost\SQLEXPRESS
    SQL_DATABASE=SistemaTickets
    SQL_USER=your_db_user
    SQL_PASSWORD=your_db_password

    JWT_SECRET=replace-with-a-strong-random-secret
    JWT_EXPIRES=8h

    SOPORTE_CODE=replace-with-a-strong-random-code

    SMTP_HOST=smtp.gmail.com
    SMTP_PORT=587
    SMTP_USER=you@gmail.com
    SMTP_PASSWORD=your_app_password
    SMTP_FROM=you@gmail.com
    SUPPORT_EMAIL=support-team@example.com
    ```

    See [Configuration](/configuration) for a full description of every variable.
  </Step>

  <Step title="Start the server">
    ```bash
    npm start
    ```

    You should see:

    ```
    ✓ Servidor en puerto 3000
    ```

    The server is now listening on `http://localhost:3000`.

    <Tip>
      You can also run `node server.js` directly. The `npm start` script calls the same command.
    </Tip>
  </Step>

  <Step title="Register a user">
    Create a new account by sending a `POST` request to `/auth/register`:

    ```bash
    curl -X POST http://localhost:3000/auth/register \
      -H "Content-Type: application/json" \
      -d '{
        "nombre": "Jane Doe",
        "correo": "jane@example.com",
        "password": "mypassword"
      }'
    ```

    A successful response returns a JWT token immediately:

    ```json
    {
      "success": true,
      "message": "Usuario registrado e identificado",
      "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
      "user": {
        "nombre": "Jane Doe",
        "rol": "USUARIO",
        "correo": "jane@example.com"
      }
    }
    ```

    <Note>
      Omitting `codigoSoporte` assigns the default `USUARIO` role. To register as `SOPORTE`, include `"codigoSoporte": "<your SOPORTE_CODE>"` in the request body.
    </Note>
  </Step>

  <Step title="Log in and get a JWT token">
    If you already have an account, log in to retrieve a fresh token:

    ```bash
    curl -X POST http://localhost:3000/auth/login \
      -H "Content-Type: application/json" \
      -d '{
        "correo": "jane@example.com",
        "password": "mypassword"
      }'
    ```

    Response:

    ```json
    {
      "success": true,
      "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
      "user": {
        "nombre": "Jane Doe",
        "rol": "USUARIO",
        "correo": "jane@example.com"
      }
    }
    ```

    Copy the `token` value — you'll include it as a `Bearer` token in the next step.
  </Step>

  <Step title="Submit a support ticket">
    Create a ticket via `POST /api/admin/tickets`:

    ```bash
    curl -X POST http://localhost:3000/api/admin/tickets \
      -H "Content-Type: application/json" \
      -d '{
        "idDep": 1,
        "nombreContacto": "Jane Doe",
        "correoContacto": "jane@example.com",
        "descripcionProblema": "My computer won'\''t turn on"
      }'
    ```

    A successful response:

    ```json
    {
      "success": true,
      "mensaje": "Ticket creado exitosamente",
      "ticket": {
        "idTicket": 42,
        "idDep": 1,
        "nombreDepartamento": "IT",
        "nombreContacto": "Jane Doe",
        "correoContacto": "jane@example.com",
        "descripcionProblema": "My computer won't turn on",
        "estado": "Abierto",
        "fechaCreacion": "2026-04-07T15:30:00.000Z"
      }
    }
    ```

    The API also sends a confirmation email to `correoContacto` and an alert to the address configured in `SUPPORT_EMAIL`.
  </Step>
</Steps>
