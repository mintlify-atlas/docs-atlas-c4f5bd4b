## Overview

API Tickets provides a backend service for creating, tracking, and resolving support tickets. Users can submit tickets describing their problems, and support staff can manage and update their status through a protected admin interface.

The API handles the full ticket lifecycle: submission, email confirmation, status updates, and closure notifications.

## Technology stack

| Component | Technology |
|---|---|
| Framework | Express.js v5 |
| Database | Microsoft SQL Server (MSSQL) |
| Authentication | JSON Web Tokens (JWT) via `jsonwebtoken` |
| Password hashing | bcrypt |
| Email | Nodemailer |
| Runtime | Node.js |

## Route groups

The API exposes three route groups:

- **`/auth`** — Registration and login. Returns a JWT token on success.
- **`/api/public`** — Unauthenticated endpoints. Lists departments and retrieves ticket status by ID.
- **`/api/admin`** — Authenticated endpoints. Requires a valid JWT in the `Authorization` header. Used to create tickets and manage their status.

## User roles

There are two roles in the system:

- **`USUARIO`** — The default role assigned on registration. Can submit support tickets. Anyone can look up a ticket by ID via the public endpoint.
- **`SOPORTE`** — Support staff role. Requires a secret `SOPORTE_CODE` at registration and login. Can view all tickets, update their status, and access ticket history.

<Note>
The `SOPORTE_CODE` is set in your `.env` file and must be shared only with support staff. See [Configuration](/configuration) for details.
</Note>
