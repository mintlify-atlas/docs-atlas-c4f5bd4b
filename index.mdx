API Tickets is an Express.js REST API for managing IT support tickets. It provides structured workflows for end users to submit requests, and for support staff to track and resolve them — with automatic email notifications at each step.

## How it works

<Steps>
  <Step title="Register or log in">
    Create an account via `POST /auth/register` or authenticate with `POST /auth/login` to receive a JWT token.
  </Step>
  <Step title="Submit a ticket">
    Call `POST /api/admin/tickets` with the department, contact name, email, and problem description. The submitter receives an email confirmation automatically.
  </Step>
  <Step title="Support team manages tickets">
    Users with the `SOPORTE` role can list all tickets, update their status (e.g., from `Abierto` to `En Proceso` to `Terminado`), and view the full history log.
  </Step>
  <Step title="Automatic notifications">
    When a ticket is marked as `Terminado`, the contact receives an automatic email confirming the resolution.
  </Step>
</Steps>

## Key features

<CardGroup cols={2}>
  <Card title="JWT Authentication" icon="shield-halved">
    Stateless token-based auth with 8-hour expiry. Tokens carry user ID, role, and email.
  </Card>
  <Card title="Role-based access" icon="user-shield">
    Two roles: `USUARIO` for general access and `SOPORTE` for ticket management. Role is enforced at the middleware level.
  </Card>
  <Card title="Email notifications" icon="envelope">
    Nodemailer sends ticket confirmation to submitters, alerts to the support inbox, and resolution notices when tickets close.
  </Card>
  <Card title="Public ticket lookup" icon="magnifying-glass">
    Anyone can look up a ticket by ID via the public endpoint — no authentication required.
  </Card>
</CardGroup>
